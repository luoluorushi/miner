## 阶段一：批量初筛

### 目标
从650+只股票中，通过财务数据快速筛选出值得深入分析的候选股。

### 输入
- `ai_stock_pool_v2.json` - 核心股票池

### 输出
- 按财务+估值评分排序的列表

### 执行步骤

```bash
# 伪代码流程
for each stock in stock_pool:
    # 1. 获取财务数据
    finance_data = stock-data finance {code} summary
    
    # 2. 获取实时行情（含估值）
    quote_data = stock-data quote {code}
    
    # 3. 计算财务质量评分
    financial_score = calculate_financial_score(
        roe = finance_data.roe_weighted,
        gross_margin = finance_data.毛利率,
        profit_growth = finance_data.jrl_tb
    )
    
    # 4. 计算估值评分
    valuation_score = calculate_valuation_score(
        pe = quote_data.pe_ratio,
        pb = quote_data.pb_ratio,
        profit_growth = finance_data.jrl_tb
    )
    
    # 5. 初筛得分 = 财务*0.6 + 估值*0.4
    prelim_score = financial_score * 0.6 + valuation_score * 0.4
    
    # 6. 写入数据库
    python db.py add {code} {name} {sector}
    python db.py update {code} -d financial_quality -s {financial_score}
    python db.py update {code} -d valuation -s {valuation_score}
```

### 财务质量评分规则

| ROE | 毛利率 | 净利润增速 | 评分 |
|-----|-------|-----------|-----|
| >20% | >40% | >50% | 5 |
| 15-20% | 30-40% | 30-50% | 4 |
| 10-15% | 20-30% | 15-30% | 3 |
| 5-10% | 10-20% | 0-15% | 2 |
| <5% | <10% | <0% | 1 |

### 估值评分规则

| PEG | PE vs 行业 | 评分 |
|-----|-----------|-----|
| <0.5 | 显著低于 | 5 |
| 0.5-0.8 | 低于 | 4 |
| 0.8-1.2 | 持平 | 3 |
| 1.2-2.0 | 高于 | 2 |
| >2.0 | 显著高于 | 1 |

## 阶段二：深度分析

### 目标
对初筛Top 100进行AI含量、护城河、需求增长等维度的深度分析。

### 输入
- 初筛排序列表Top 100
- stock-data工具
- Web搜索能力

### 输出
- 完整的7维度评估数据
- AI总评分

### 执行步骤（以单只股票为例）

#### Step 1: 收集脚本可获取的数据
```bash
# 公司简况
stock-data profile {code}

# 财务数据
stock-data finance {code} summary

# 实时行情
stock-data quote {code}

# 研报评级
stock-data rating {code}

# 最新新闻
stock-data news {code}
```

#### Step 2: 搜索补充信息（LLM执行）

**AI含量搜索**：
```
- "{公司名称} AI业务收入占比"
- "{公司名称} 英伟达/微软/谷歌 供应商"
- "{公司名称} AI服务器/数据中心 订单"
- "{公司名称} 2025年报 AI业务"
```

**护城河搜索**：
```
- "{公司名称} 市占率 全球排名"
- "{公司名称} 核心技术 专利"
- "{公司名称} 竞争优势 vs {竞争对手}"
```

**需求增长搜索**：
```
- "{所在行业} 市场规模 增速 2026"
- "{公司名称} 订单 产能利用率 扩产"
- "{下游行业} 资本开支 2026"
```

**催化剂/风险搜索**：
```
- "{公司名称} 最新利好 新产品"
- "{公司名称} 风险 减持 解禁"
```

#### Step 3: 计算市值/AI利润比
```bash
# 获取市值和利润
market_cap = quote_data.market_cap
profit = finance_data.profit_ttm

# AI收入占比从搜索结果获取
ai_revenue_pct = 85  # 例如85%

# 计算
ai_profit = profit * ai_revenue_pct / 100
mcap_ai_profit_ratio = market_cap / ai_profit

# 评分
if mcap_ai_profit_ratio < 30:
    score = 5
elif mcap_ai_profit_ratio < 50:
    score = 4
elif mcap_ai_profit_ratio < 80:
    score = 3
elif mcap_ai_profit_ratio < 150:
    score = 2
else:
    score = 1
```

#### Step 4: 更新数据库
```bash
# 更新各维度评分
python db.py update {code} -d ai_exposure -s 5 -e "AI收入占比85%"
python db.py update {code} -d moat -s 5 -e "全球800G市占率第一"
python db.py update {code} -d demand_growth -s 5 -e "行业CAGR 60%"
python db.py update {code} -d mcap_ai_profit_ratio -s 2 -e "市值/AI利润100x"
python db.py update {code} -d catalyst_risk -s 4 -e "1.6T放量"

# 查看更新后的结果
python db.py get {code}
```

## 阶段四：持续维护

### 触发条件

| 事件 | 动作 |
|-----|------|
| 新股加入股票池 | `python db.py add` + 触发完整评估流程 |
| 季报发布 | 更新财务数据和评分 |
| 重大公告 | 更新催化剂/风险评分 |
| 股价大幅波动 | 更新估值评分 |
| 用户要求 | 手动触发重新评估 |

### 数据更新频率

| 维度 | 更新频率 | 更新方式 |
|-----|---------|---------|
| 财务质量 | 季度 | 脚本自动 |
| 估值 | 每日 | 脚本自动 |
| AI含量 | 半年度 | LLM分析 |
| 护城河 | 年度 | LLM分析 |
| 需求增长 | 季度 | LLM分析 |
| 市值/AI利润比 | 季度 | 脚本计算 |
| 催化剂/风险 | 实时/事件驱动 | LLM分析 |
